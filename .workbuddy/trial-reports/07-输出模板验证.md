## 验证报告 — 输出模板格式

**验证时间**：2026-04-22
**验证文件**：output-template.md、distillation-sop.md、quality-control.md、SKILL.md
**验证角色**：模板验证-G（输出模板格式验证专家）

---

### 验证项1：单事件模板完整性

**必填字段覆盖率：7/7（100%）**

| SKILL.md 要求的必填内容 | 模板中对应字段 | 状态 |
|---|---|---|
| 事件概况 | §事件概况（30秒速览）：时间、领域、关键角色、核心驱动力、最终结果 | ✅ 有 |
| 核心规律 | §可复用规律：规律标题、描述、来源、适用/不适用、验证 | ✅ 有 |
| 信任度评级 | §信任度评级：4个维度+综合评级 | ✅ 有 |
| 一句话结论（≤30字） | 引用块 `> 一句话结论：[...]` | ✅ 有 |
| 适用/不适用场景 | §适用场景 + §不适用场景（带原因） | ✅ 有 |
| 分析框架选择 | §事件结构化分析 > 分析框架 | ✅ 有 |
| 关键决策链 | §关键决策链（含反事实推演） | ✅ 有 |

**详细发现**：

1. **一句话结论位置合理**：放在模板最顶部引用块中，用户一眼就能看到核心教训。标注了"不超过30字"的约束。
2. **适用/不适用场景是事件级别的**：不是按规律标注的，而是整个蒸馏结果的适用边界。这与SKILL.md中"每条规律标注适用边界"的要求形成两层设计——规律级边界在"可复用规律"中标注，事件级边界在"适用/不适用场景"中标注。设计合理。
3. **SOP第3步"决策拆解"字段覆盖**：
   - 决策时间 → 模板有 ✅
   - 决策者 → 模板有 ✅
   - 决策内容 → 模板有（"面临的选择"） ✅
   - 当时信息 → 模板有（"当时信息"） ✅
   - 面临压力 → 模板**缺失** ⚠️
   - 可选方案 → 模板有（方案A/B） ✅
   - 选择理由 → 模板有 ✅
   - 实际结果 → 模板有 ✅
   - 预期vs实际 → 模板**缺失** ⚠️

**结论**：核心必填字段全覆盖。SOP中定义的2个决策分析字段（"面临压力"和"预期vs实际"）在模板中缺失，但这两个是深度分析细节，简要版中不包含可接受，深度版中应考虑补充。

### 验证项2：多事件模板完整性

**字段覆盖检查**：

| 检查项 | 状态 | 说明 |
|---|---|---|
| 事件概览表字段与单事件一致 | ✅ 一致 | 概览表有：事件名称、事件类型、核心驱动力、最终结果。单事件概况有：时间、领域、关键角色、核心驱动力、最终结果。**不完全一致**——概览表缺少"时间"和"关键角色"，但这是合理的简化，概览表的目的是快速对比多个事件，字段过多会导致表格臃肿 |
| 共识规律板块 | ✅ 完整 | 包含：规律标题、描述、验证事件、适用条件、失效条件 |
| 矛盾发现板块 | ✅ 完整 | 包含：事件A/B的观点、矛盾原因、给你的启示 |
| 条件边界板块 | ✅ 完 | 表格形式：规律、成立条件、失效条件、判断方法 |
| 信任度评级维度 | ✅ 与单事件不同 | 单事件：信息来源、多方验证、因果清晰度、规律可靠性。多事件：事件覆盖度、跨类型验证、规律可靠性、信息来源 |

**发现**：

1. **"矛盾发现"中的"给你的启示"设计好**：矛盾发现不只是列出矛盾，还要求给出用户视角的判断指引，提升了实用性。
2. **多事件模板缺少"各事件关键教训"的结构化程度**：当前"各事件关键教训"部分只写了"[简要蒸馏：核心决策 + 教训 + 与主题的关联]"，没有给出具体字段结构，AI自由发挥空间过大。建议给出至少：核心决策、关键教训、与本主题的关联三个明确的子字段。
3. **多事件模板没有"一句话结论"**：单事件模板顶部有"一句话结论"，多事件模板顶部是"核心发现"。两者命名不同但功能相同（快速传达核心结论），无问题。

### 验证项3：反向蒸馏模板完整性

| 检查项 | 状态 | 说明 |
|---|---|---|
| "你的处境"板块 | ✅ 有 | 包含：行业/领域、当前阶段、核心困境、可用资源 |
| 相似点 | ✅ 有 | "与你的相似点"明确列出 |
| 差异点 | ✅ 有 | "与你的差异点"明确列出 |
| 差异点标注"对用户的影响" | ✅ 有 | 格式为"差异点1 — 这意味着 [对你的影响]" |
| 综合启示板块 | ✅ 有 | "综合启示"列出了3条启示 |
| 免责声明 | ✅ 有 | "以上是基于历史案例的启示，不是对你的决策建议。你的具体环境与历史案例存在差异，最终决策需要结合自身情况判断。" |
| 信任度评级 | ❌ **缺失** | 反向蒸馏模板没有信任度评级板块 |

**发现**：

1. **反向蒸馏模板缺少信任度评级**：这是一个显著问题。SKILL.md要求"每个输出必须带信任度评级"，但反向蒸馏模板没有包含。考虑到反向蒸馏依赖搜索匹配质量（SKILL.md自己也说"匹配质量低时应主动告知用户"），信任度评级更加重要——用户需要知道匹配到的案例与自己的处境有多大关联度。
2. **反向蒸馏模板缺少"降级处理"的输出指引**：SKILL.md定义了降级处理规则（案例少于3个或相关度低时告知用户），但模板中没有给出降级输出格式。AI可能不知道如何格式化降级提示。
3. **"综合启示"的编号方式是开放的**：写的是"1/2/3"，没有限制条数。考虑到反向蒸馏匹配3个事件，2-3条启示是合理的，建议标注"2-3条"。

### 验证项4：简要版 vs 深度版

| 检查项 | 状态 | 说明 |
|---|---|---|
| 简要版字段清单明确 | ✅ 明确 | 列出了4项：事件概况、一句话结论、2-3条核心规律（带适用边界）、信任度评级 |
| 深度版字段清单明确 | ✅ 明确 | 列出了6项：完整决策链、反事实推演、多方验证细节、规律交叉验证案例、适用/不适用详细说明、不限字数 |
| 切换机制明确 | ✅ 明确 | "先输出简要版，再询问用户是否需要深度版" |

**发现**：

1. **简要版字段覆盖了SKILL.md的核心要求**：一句话结论、核心规律（带适用边界）、信任度评级——三个SKILL.md明确要求的输出项都在简要版中。
2. **深度版没有包含"事件结构化分析"**：深度版的字段清单中没有明确列出"事件结构化分析"（事件背景+分析框架+关键转折），但完整决策链和分析框架是深度版应有的内容。建议在深度版清单中明确加入。
3. **简要版中"信任度评级"在单事件模板中内容较重**：简要版包含4个维度的信任度评级表，对于"简要"来说信息密度偏高。可以考虑简要版使用综合评级+一句话说明，深度版使用4维度表格。但这是一个优化建议而非缺陷。

### 验证项5：模板实用性

**对AI的可用性**：

| 评估维度 | 评分 | 说明 |
|---|---|---|
| 字段名明确度 | 优秀 | 每个字段都有明确的名称和格式（如"时间"、"领域"、"核心驱动力"），AI可以直接填入 |
| 填写说明清晰度 | 优秀 | 有括号说明和示例，如"[一句话概括驱动事态发展的关键因素，不是'发生了什么'，而是'什么驱动了发展']"——这种否定式说明有效防止AI偏离 |
| 自由发挥空间控制 | 良好 | 大部分字段都有约束说明，但"各事件关键教训"和"综合启示"相对开放 |
| 格式一致性 | 优秀 | 三种模板格式风格统一，都使用markdown标题+列表/表格 |

**对用户的可读性**：

| 评估维度 | 评分 | 说明 |
|---|---|---|
| 信息层级 | 优秀 | 一句话结论→概况→分析→决策链→规律→评级，从速览到深入的递进逻辑清晰 |
| 避免纯表格堆砌 | 良好 | 混合使用了表格和列表，但"事件背景"和"关键转折"都是纯表格形式，连续阅读时可能产生视觉疲劳 |
| 快速定位能力 | 优秀 | "30秒速览"、"适用/不适用场景"等标题帮助用户快速定位关心部分 |

**信任度评级位置**：

| 模板类型 | 位置 | 评估 |
|---|---|---|
| 单事件 | 模板末尾 | ✅ 合理，用户看完分析后判断可靠性 |
| 多事件 | 模板末尾 | ✅ 合理 |

### 问题汇总

| 严重程度 | 模板类型 | 问题描述 | 建议 |
|---|---|---|---|
| **P0 高** | 反向蒸馏 | 缺少信任度评级板块，违反SKILL.md"每个输出必须带信任度评级"的要求 | 新增信任度评级板块。反向蒸馏的评级维度建议调整为：匹配相关度、信息充分度、案例多样性、规律可靠性 |
| **P1 中** | 反向蒸馏 | 缺少降级处理输出格式。SKILL.md定义了降级规则但模板中没有对应格式 | 新增降级输出模板片段，当匹配案例不足或相关度低时使用 |
| **P1 中** | 单事件 | SOP定义了"面临压力"和"预期vs实际"两个决策分析字段，但模板中缺失。深度版应包含这两个字段 | 在深度版字段清单中明确加入这两个字段，或在决策链模板中补充 |
| **P2 低** | 多事件 | "各事件关键教训"部分缺少结构化字段，AI自由发挥空间过大 | 补充结构化格式：核心决策[1句话]、关键教训[1-2条]、与主题关联[1句话] |
| **P2 低** | 单事件 | 简要版的信任度评级（4维度表格）信息密度偏高，与"简要"定位不完全匹配 | 考虑简要版用综合评级+一句话说明，深度版用完整维度表格 |
| **P2 低** | 所有模板 | 模板中没有自检提示（如"检查是否有事后诸葛亮语言"），质控检查与输出模板是分离的 | 可在模板末尾添加一个隐藏的"AI自检清单"，不输出给用户，但提醒AI在输出前自查。这是一个可选优化 |

### 亮点（做得好的地方）

1. **一句话结论设计精炼**：放在模板最顶部，≤30字约束，让用户3秒内抓住核心教训。这是整个模板中最有用户价值的元素。

2. **规律的三层边界标注**：来源（回溯到具体事件决策节点）→ 适用/不适用（场景级边界）→ 验证强度（样本量级），构成了从微观到宏观的完整验证链条。这是防鸡汤化的有效设计。

3. **"核心驱动力"的否定式说明**："[一句话概括驱动事态发展的关键因素，不是'发生了什么'，而是'什么驱动了发展']"——这种"不是X而是Y"的说明方式能有效防止AI输出流水账式的事件概述。

4. **多事件信任度评级维度差异化**：单事件用"信息来源/多方验证/因果清晰度/规律可靠性"，多事件用"事件覆盖度/跨类型验证/规律可靠性/信息来源"。维度根据蒸馏模式调整，不是机械复制，设计合理。

5. **适用/不适用场景的双层设计**：事件级适用边界（独立板块）+ 规律级适用边界（每条规律下），既有宏观判断又有微观校验。

6. **反事实推演的约束设计**：模板中"如果选另一个"字段配合SOP中的硬性约束（替代方案必须是当时决策者真正考虑过的），有效防止AI开脑洞式推演。

7. **简要版/深度版的分流机制**："先输出简要版，再询问用户是否需要深度版"——默认轻量、按需深入，符合用户使用效率。
