# 事件类型搜索策略指南

## 概述

不同事件类型需要不同的搜索关键词和来源策略。本文档为5类事件分别定义搜索方案。

## 搜索的局限

本指南定义的搜索策略能显著提升信息获取质量，但搜索本身存在以下固有局限：

- **公开资料≠完整信息**：所有搜索基于公开可获取的信息。内部决策记录、未公开数据、仍在保密期内的信息无法通过搜索获取。蒸馏结果的可靠性上限由公开资料的充分度决定
- **搜索引擎的覆盖偏差**：英语事件的公开资料远多于其他语言；主流叙事的搜索结果远多于少数派观点；近期事件的资料可能不完整（还在发展中或尚未被深度分析）
- **历史记载的立场偏差**：史书由胜者书写，现代媒体有政治/商业立场，当事人自述有美化动机。本指南要求多源交叉验证，但如果所有来源都偏向同一叙事，交叉验证也无法纠正
- **优先来源是示例不是穷举**：各事件类型下列出的优先来源是经验性的质量排序，不代表这些是唯一的可靠来源。其他来源如果满足可信度标准（可追溯、有细节、可交叉验证），同样可以使用

## 1. 公众商业事件

### 搜索关键词模板

| 搜索角度 | 关键词模式 | 示例 |
|---------|----------|------|
| 事件经过 | [事件名] + 时间线/经过/始末 | 「Ofo 破产 时间线」 |
| 决策内幕 | [事件名] + 决策/内幕/复盘/反思 | 「WeWork 估值暴跌 决策失误」 |
| 第三方分析 | [事件名] + 分析/深度/案例研究 | 「诺基亚 被微软收购 深度分析」 |
| 当事人视角 | [人物名] + 访谈/传记/自述/播客 | 「戴威 ofo 创始人 访谈」 |
| 财务数据 | [公司名] + 财报/年报/招股书 | 「WeWork 招股书 风险因素」 |

### 优先来源
- 财报/招股书（SEC文件）
- 财经深度媒体（财新、Bloomberg长文、Wired、The Information）
- 创始人传记/播客访谈（How I Built This、Acquired播客）
- 券商/咨询公司行业报告

### 注意事项
- 当代事件的最终结果可能尚未完全显现，蒸馏时需要标注
- 创始人访谈可能美化自己的决策，必须交叉验证
- 注意事件发生时的宏观经济背景（同一决策在不同经济周期效果不同）

## 2. 历史事件

### 搜索关键词模板

| 搜索角度 | 关键词模式 | 示例 |
|---------|----------|------|
| 事件经过 | [事件名] + 经过/始末/背景 | 「鸿门宴 经过 背景」 |
| 决策分析 | [事件名] + 决策/策略/博弈 | 「赤壁之战 战略决策 分析」 |
| 学术研究 | [事件名] + 论文/研究/史学 | 「郑伯克段于鄢 史学论文」 |
| 不同解读 | [事件名] + 不同观点/争议/新解 | 「玄武门之变 不同解读」 |
| 现代启示 | [事件名] + 启示/现代意义/管理 | 「商鞅变法 现代管理启示」 |

### 优先来源
- 正史（《史记》《资治通鉴》《左传》等——但要注意史书本身的政治立场）
- 现代史学研究论文（知网、JSTOR、Google Scholar）
- 历史学家的专著（不同学派的对比）
- 纪录片/历史讲座

### 注意事项
- 史书记载本身可能有政治偏见（胜者书写历史），需要多版本交叉验证
- 古代的政治/经济环境与现代差异大，提炼规律时必须标注适用条件
- 区分「史实」和「史家解读」——前者是事实，后者是观点
- 军事类历史事件的搜索可与第4类（军事事件）策略合并

## 3. 科技/工程事件

### 搜索关键词模板

| 搜索角度 | 关键词模式 | 示例 |
|---------|----------|------|
| 事故报告 | [事件名] + 调查报告/事故报告/官方报告 | 「波音737 MAX 事故调查报告」 |
| 技术分析 | [事件名] + 技术分析/工程失误/根本原因 | 「福岛核泄漏 根本原因分析」 |
| 组织决策 | [事件名] + 组织决策/管理失误/文化 | 「挑战者号 NASA 组织决策」 |
| 后续改进 | [事件名] + 教训/改进/后续 | 「三星Note7 召回 教训改进」 |

### 优先来源
- 官方事故调查报告（NTSB、FAA、NASA等）——**这是最高质量来源**
- 工程期刊/技术分析文章
- IEEE Spectrum、MIT Technology Review等工程技术媒体
- 纪录片（如《挑战者号：最后的飞行》）

### 注意事项
- 事故调查报告本身就是结构化的决策分析，可直接作为蒸馏框架
- 技术事件有时涉及专有信息或保密内容，公开信息可能不完整
- 注意区分「技术失败」和「组织决策失败」——很多工程事故的根因是后者

## 4. 军事事件

### 搜索关键词模板

| 搜索角度 | 关键词模式 | 示例 |
|---------|----------|------|
| 战役经过 | [战役名] + 经过/时间线/兵力 | 「诺曼底登陆 经过 时间线」 |
| 战略决策 | [战役名] + 战略/决策/计划 | 「珍珠港事件 日本决策过程」 |
| 军事分析 | [战役名] + 军事分析/战术/战略 | 「赤壁之战 战略分析」 |
| 情报决策 | [事件名] + 情报/误判/信息战 | 「珍珠港 情报误判」 |
| 后勤/资源 | [事件名] + 后勤/补给/资源 | 「马岛战争 后勤补给」 |

### 优先来源
- 军事学院教材/战史专著
- 解密军事档案
- 军事史学术研究
- 纪录片/历史频道

### 注意事项
- 现代军事事件可能涉及保密信息，公开信息可能不完整
- 军事与商业的类比有边界：军事是零和博弈，商业可以是正和博弈。蒸馏时必须标注
- 军事决策的政治维度复杂，蒸馏聚焦决策逻辑，不评价战争本身

## 5. 政策/法规事件

### 搜索关键词模板

| 搜索角度 | 关键词模式 | 示例 |
|---------|----------|------|
| 政策内容 | [政策名] + 全文/解读/要点 | 「双减政策 全文 解读」 |
| 制定过程 | [政策名] + 制定过程/背景/起因 | 「GDPR 制定过程 背景」 |
| 影响分析 | [政策名] + 影响/行业/企业应对 | 「双减 教培行业 影响」 |
| 国际对比 | [政策名] + 国际对比/其他国家 | 「数据安全法 国际对比」 |
| 政策演变 | [政策名] + 演变/修订/历史 | 「禁酒令 颁布 废除 过程」 |

### 优先来源
- 政策原文（政府官网、官方公报）
- 权威解读（国务院解读、欧盟委员会FAQ等）
- 行业影响分析报告（券商、咨询公司）
- 学术政策研究

### 注意事项
- 政策事件的蒸馏聚焦**决策逻辑和企业应对**，不做政治评价
- 政策在不同国家/地区的逻辑差异大，蒸馏时标注地域背景
- 政策效果往往有滞后性，需要区分短期反应和长期影响
- 注意区分「政策制定者的决策」和「受影响企业的决策」——两者可能都需要蒸馏

## 跨类型事件的处理

很多事件跨越多个类型，处理规则：

| 情况 | 处理方式 |
|------|---------|
| 商业 + 科技/工程（如波音737 MAX） | 以商业决策为框架，技术分析为支撑 |
| 历史 + 军事（如赤壁之战） | 以历史事件搜索策略为主，补充军事分析 |
| 政策 + 商业（如教培双减） | 分别蒸馏政策制定逻辑和企业应对逻辑 |
| 跨所有类型（如新冠疫情期间的企业应对） | 根据用户关心的维度选择主导类型 |

**原则**：跨类型事件不要试图全部覆盖，根据用户意图选择主导类型，其他类型作为补充信息。
