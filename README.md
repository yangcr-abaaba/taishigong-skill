# 太史公 — 事件决策蒸馏

将真实事件的决策过程提炼为可复用的结构化决策参考。不是事件概述，不是新闻报道，是「从别人的决策中提炼自己能用的框架」。

## 为什么做这个 Skill

AI 最擅长的事情之一是「给你建议」。但你发现没有，这些建议大多数没什么用——它们要么是正确的废话（「要聚焦核心业务」），要么是事后诸葛亮（「他们当时应该做 X」），要么是一堆上中下策让你自己选。真正有用的不是建议，而是前人在类似处境下做了什么、为什么这么做、以及结果是什么。

太史公的灵感来自一个朴素的想法：**历史和商业事件里藏着大量的决策样本，但它们散落在文章、传记、案例研究里，没有被结构化地提炼成可复用的参考。** AI 可以做这件事——但它需要被严格约束，否则它会本能地滑向「总结 + 建议」的模式，产出跟一篇普通的分析文章没什么区别。

所以这个 Skill 的核心设计不是「让 AI 更聪明」，而是**让 AI 管住自己**：不评价人、不替你选、不编造、不预设立场，只做一件事——从公开资料中还原决策链，提炼规律，标注这条规律在什么条件下可能适用、什么条件下会失效。它给自己的定位是「蒸馏者」而不是「参谋」。

还有一个更底层的思考：人使用任何工具和技能，最终都是为了更好地处理自己面对的事件。理解一个人可以蒸馏人，理解一本书可以蒸馏书，但这些蒸馏结果最终都要落到「怎么处理这件事」上。既然终点是事件，为什么不直接蒸馏事件本身？

太史公就是这个答案。它只蒸馏事件——把真实发生过的决策过程拆解成结构化的参考卡片，让你在面对自己的处境时，能拿前人的决策样本做对照，而不是听一堆正确的废话。

名字叫「太史公」，是因为司马迁写的不是谏书，是史记。记录发生了什么、为什么发生，把判断留给后人。

## 它做什么

你给它一个事件（或一个决策困境），它还你一张**决策参考卡片**——拆解了关键决策节点、反事实推演（如果选了另一条路会怎样）、可复用规律、适用边界和信任度评级。

**三种使用方式：**

| 模式 | 你怎么说 | 它做什么 |
|------|---------|---------|
| 单事件蒸馏 | 「帮我分析Ofo的崩溃」「诺基亚为什么衰败」 | 深度拆解一个事件的决策逻辑 |
| 多事件蒸馏 | 「企业面对颠覆性技术该怎么决策」 | 匹配多个相关事件，交叉验证规律 |
| 反向蒸馏 | 「我现在面临转型压力，团队也在内耗」 | 从你的处境出发，匹配类似历史案例对比分析 |

## 覆盖范围

**5类事件**：公众商业事件、历史事件、科技/工程事件、军事事件、政策/法规事件

**6个决策维度**：竞争博弈、战略选择、危机应对、政策影响、经验学习、反向蒸馏

**不触发**：纯操作指南（"怎么写简历"）、政治人物评价、尚无定论的争议事件

## 工作原理

```
意图识别 → 信息采集 → 结构化分析 → 决策拆解 → 反事实推演 → 规律提炼 → 质量控制 → 输出
```

### 结构化分析：4种分析框架按需选用

不是每个事件都套同一个框架。根据事件性质选择最贴合的分析工具：

| 框架 | 适用于 | 不适用于 |
|------|--------|---------|
| **A. 矛盾分析法** | 利益冲突明确的商业竞争、政治博弈、军事对抗 | 技术事故、政策制定 |
| **B. 决策链分析法** | 决策失误/成功驱动的事件（挑战者号、柯达） | 多因素系统性行为 |
| **C. 因果链分析法** | 多因素共同导致的结果（金融危机） | 单一决策失误 |
| **D. 利益相关方分析法** | 多方博弈（GDPR制定、政策制定） | 双方对抗 |

复杂事件可组合1-2个框架，但不堆砌。

### 质量控制：五要素验证

每条输出必须通过：

1. 防事后诸葛亮 — 还原决策者当时的信息不完备状态，禁用「显然应该…」
2. 防幸存者偏差 — 成功事件强制找反例，失败事件强制找成功先例
3. 防信息源偏差 — 至少3个不同立场的来源交叉验证
4. 防过度抽象 — 每条规律必须能回溯到具体事件细节，有适用边界
5. 中立性原则 — 规律描述使用机制语言，不使用价值判断或心理归因语言

## 输出格式

默认输出**简要版**（事件概况 + 核心规律 + 信任度评级），需要时可展开**深度版**（完整决策链 + 反事实推演 + 交叉验证）。

每条规律标注适用条件和失效条件。每个输出附带信任度评级（高/中/低），低信任度时会建议补充信息源。

## 能力边界

**能做到：** 基于公开信息拆解决策逻辑、提炼可复用规律、交叉验证、标注信任度

**做不到的：**
- 获取非公开信息（内部决策记录、未公开财务数据等）
- 验证事实真伪（依赖搜索结果，交叉验证能降低但不能消除偏差）
- 反事实推演不是预测（推测≠可靠预测）
- 信任度评级是主观判断（高/中/低是参考信号，不是精确度量）
- 规律不是定律（从历史提炼的规律是概率性模式，不是必然成立的规律）
- 搜索受限于搜索引擎（非英语事件、小众事件、新近事件的资料可能不足）
- 替代专业领域研究（医学、法律、精密工程等需要专业知识的领域）
- 保证完全客观中立（搜索结果的来源分布本身可能有偏向）
- 质控不能替代用户判断（通过质控检查不等于结论正确）

**使用建议：** 把蒸馏结果当作「有结构的思考起点」，而不是「权威结论」。信任度评级告诉你能信多少，但不能替代你自己的判断。

## 关键约束

- 不评价政治人物 — 可以分析政治事件的决策逻辑，但不评价人
- 不替用户做决定 — 输出是决策参考，不是决策建议
- 不编造信息 — 无法验证的信息必须标注
- 不强求通用性 — 提炼不出有价值的规律就如实告知
- 不预设立场 — 事实指向什么结论就输出什么结论
